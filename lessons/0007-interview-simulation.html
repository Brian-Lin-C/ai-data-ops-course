🎤 Lesson 7: 面试模拟 — AI数据运营策略岗

💡 核心洞察：这个岗位的面试有 4 类题——SQL 实战、案例分析、大模型认知、项目管理。每类题的答题框架和踩坑点都不同。

2. SQL 题 — 三道必练

① 窗口函数：每个品类销售额 Top 3 商品

SELECT category, product_name, total_sales
FROM (
  SELECT category, product_name, 
    SUM(amount) AS total_sales,
    ROW_NUMBER() OVER (PARTITION BY category ORDER BY SUM(amount) DESC) AS rn
  FROM orders
  WHERE dt >= DATE_SUB(CURDATE(), INTERVAL 1 MONTH)
  GROUP BY category, product_name
) t
WHERE rn <= 3;

② 留存计算：次日/7日/30日留存

WITH first AS (
  SELECT user_id, MIN(DATE(login_time)) AS first_dt
  FROM login GROUP BY user_id
)
SELECT first_dt,
  COUNT(DISTINCT f.user_id) AS new_users,
  ROUND(COUNT(DISTINCT CASE WHEN DATEDIFF(l.login_time,f.first_dt)=1 THEN f.user_id END)*100.0/COUNT(DISTINCT f.user_id),2) AS d1,
  ROUND(COUNT(DISTINCT CASE WHEN DATEDIFF(l.login_time,f.first_dt)=6 THEN f.user_id END)*100.0/COUNT(DISTINCT f.user_id),2) AS d7,
  ROUND(COUNT(DISTINCT CASE WHEN DATEDIFF(l.login_time,f.first_dt)=29 THEN f.user_id END)*100.0/COUNT(DISTINCT f.user_id),2) AS d30
FROM first f LEFT JOIN login l ON f.user_id=l.user_id
GROUP BY first_dt;

③ 连续登录：连续登录 ≥3 天的用户

WITH t AS (
  SELECT DISTINCT user_id, DATE(login_time) AS dt FROM login
),
t2 AS (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY dt) AS rn
  FROM t
)
SELECT DISTINCT user_id
FROM t2
GROUP BY user_id, DATE_SUB(dt, INTERVAL rn DAY)
HAVING COUNT(*) >= 3;

3. 案例分析题 — 标准答题框架

题型：「某天 DAU 下降了 15%，你会怎么排查？」（来源：Airbnb/OpenAI 面试指南）

答题框架（STAR 变体 — DAIR）：

Define — 先确认口径：是不是数据延迟？统计口径变了？去年同期大促造成的基数效应？

Analyze — 拆维度下钻：渠道→地域→新老用户→设备→功能模块。哪个维度异常？

Identify — 定位根因：结合业务事件（竞品活动/功能上线/服务器问题），给出置信度判断

Recommend — 给出建议：短期修复方案 + 长期监控机制。量化预估影响

OpenAI 数据分析岗真实面试题（来源：DataFord Interview Guide）

「工单量在计费变更后飙升了 25%。你的排查思路是什么？」

标准回答：① 确认是否计费变更导致（时间对齐）→ ② 拆工单类型：是新用户不会用还是老用户投诉？→ ③ 区分正常增长和异常增长（看历史波动范围）→ ④ 如果确认是计费变更导致，建议增加引导流程或临时客服扩容

4. 大模型认知题 — 最关键的差异化

面试官：「你对大模型在数据分析中的应用有什么理解？」

你的回答：「我认为大模型在数据分析有三个成熟的应用场景：第一是 NL2SQL——让业务人员用自然语言查数据，AWS 和 Google 都有成熟的实践方案，核心是 Prompt Engineering + Schema 上下文。第二是自动分析报告——把数据摘要喂给 LLM，让它生成结构化的分析报告初稿，分析师做审核和深化。第三是智能数据清洗——LLM 能识别异常值模式和不一致格式，生成 pandas 清洗代码。我自己做过一个小 demo，用 DeepSeek API 实现了自然语言查订单数据的功能。」

5. 项目管理题

面试官：「你主导过一个数据项目吗？怎么确保它不烂尾？」

回答要点：

① 立项明确——目标、范围、成功标准写在一页纸上，干系人签字确认

② 里程碑驱动——拆 3-5 个关键节点，每个节点有交付物和评审

③ 风险前置——数据项目的 Top 3 风险永远是：数据不可得、口径不一致、需求变更

④ 收益量化——项目结束必须有数字：「这个看板让异常响应从 T+1 降到 T+0，每天节省 1.5 小时查数时间」

6. 面试通用原则

结构化所有答案用框架回答。不说「我觉得」，说「我先确认口径→然后拆维度→最后给建议」
量化任何「我做了 X」后面必须跟数据。多少时间？多少人？什么结果？
主动扩展回答完基础问题后，主动带出你的差异化技能（"我还用 LLM 做过类似的自动化..."）
反问环节别问「公司福利怎么样」。问「团队目前数据建设最大的挑战是什么？」「AI 在数据工作中的应用到了什么阶段？」
