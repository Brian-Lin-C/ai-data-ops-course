🤖 Lesson 5: 大模型赋能数据工作

来源: AWS Text2SQL Best Practices, LlamaIndex PandasQueryEngine, Google BigQuery NL2SQL, arXiv NL2SQL Survey 2024

💡 核心洞察：JD 写「大模型赋能数据管理」不是让你训练模型，是让你用 LLM 让数据工作自动化——自然语言查数据、AI 写分析报告、智能数据清洗。这是你面试中最强的差异化武器。

1. NL2SQL — 自然语言转 SQL（来源：AWS 官方博客）

这是大模型在数据领域最成熟的应用。AWS 总结了完整的最佳实践：

核心流程三步

① 自然语言理解 — 解析用户输入，提取关键要素（要查什么表？什么字段？什么条件？）

② SQL 生成 — LLM 将结构化意图转为可执行 SQL

③ 数据库查询 — 执行 SQL，取回结果，返回给用户

Prompt Engineering 核心（来源：AWS 最佳实践）

-- Prompt 模板：告诉 LLM 你要 SQL
/*
给定以下数据库 schema：
CREATE TABLE orders (
  order_id INT,
  user_id INT,
  amount DECIMAL,
  order_time TIMESTAMP
);
CREATE TABLE users (
  user_id INT,
  name TEXT,
  region TEXT
);

请回答以下问题，生成 MySQL 查询：
"上月华东区各品类销售额排名"
*/

SELECT 
  u.region,
  DATE_FORMAT(o.order_time, '%Y-%m') AS month,
  SUM(o.amount) AS total_sales
FROM orders o
JOIN users u ON o.user_id = u.user_id
WHERE u.region = '华东'
  AND o.order_time >= DATE_SUB(CURDATE(), INTERVAL 1 MONTH)
GROUP BY u.region, month
ORDER BY total_sales DESC;

AWS 核心建议

先 Prompt 后微调先用精心设计的 Prompt 驱动基础模型。只有 Prompt 不够时才考虑微调（需要大量标注数据）
Few-shot 示例Prompt 里放 2-3 组「自然语言→SQL」对照，模型能学得更准
RAG 增强检索最相似的 Text2SQL 示例注入 Prompt，提升生成准确度
Schema 上下文必须在 Prompt 中提供完整的表结构和字段说明

2. AI 自动分析报告

第二个核心场景：把数据扔给 LLM，让它自动生成分析报告。

# 用 LLM 自动生成数据分析报告
import openai  # 或 deepseek / qwen

data_summary = f"""
GMV: 1,250万 (环比+8.2%)
订单数: 3.4万 (环比+5.1%)
客单价: ¥368 (环比+3.0%)
退货率: 2.1% (环比-0.3pp)
Top品类: 服装(+15%), 食品(+3%), 数码(-5%)
"""

prompt = f"""你是一名资深数据分析师。以下是本月业务数据摘要：

{data_summary}

请撰写一份 200 字以内的业务分析报告，包含：
1. 总体表现评价
2. 值得关注的异常点
3. 下月行动建议
"""

response = openai.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role":"user","content":prompt}]
)
print(response.choices[0].message.content)
